# Topface Media — Data Analyst Test Task

Репозиторий с решением тестового задания на позицию Data Analyst / Junior Data Analyst на основе задания от Topface Media. Источник информации о компании: [Topface Media](https://topfacemedia.com/about).

## Описание задания

В рамках тестового задания требовалось подготовить аналитический отчёт по пролонгациям проектов сотрудников отдела сопровождения клиентов за 2023 год.

Цель анализа — оценить, насколько эффективно аккаунт-менеджеры выполняют одну из ключевых задач отдела: пролонгацию договоров с клиентами.

В работе использовались два исходных набора данных:

- **prolongations.csv**
  - `id` — идентификатор проекта
  - `month` — последний месяц реализации проекта
  - `AM` — ФИО ответственного аккаунт-менеджера

- **financial_data.csv**
  - `id` — идентификатор проекта
  - `Причина дубля` — причина, по которой один `id` встречается несколько раз
  - колонки месяцев — суммы отгрузок проекта по месяцам
  - `Account` — ФИО ответственного аккаунт-менеджера

### Метрики по ТЗ

В задании требовалось рассчитать два коэффициента пролонгации:

1. **Коэффициент пролонгации в первый месяц**  
   Отношение суммы отгрузки проектов, пролонгированных в первый месяц после завершения, к сумме отгрузки последнего месяца реализации всех проектов, завершившихся в прошлом месяце.

2. **Коэффициент пролонгации во второй месяц**  
   Отношение суммы отгрузки проектов, пролонгированных во второй месяц, к сумме отгрузки последнего месяца реализации проектов, которые не были пролонгированы в первый месяц.

Расчёты требовалось выполнить:
- по каждому аккаунт-менеджеру;
- по отделу в целом;
- помесячно за 2023 год;
- по итогам всего 2023 года.

## Какие работы были выполнены

В рамках решения были выполнены следующие этапы:

1. **Загрузка и первичная проверка данных**
   - импорт данных из `prolongations.csv` и `financial_data.csv`;
   - проверка структуры таблиц, пропусков и форматов.

2. **Подготовка данных**
   - преобразование текстовых названий месяцев в формат дат;
   - выделение месячных колонок в `financial_data.csv`;
   - очистка и нормализация значений в ячейках.

3. **Агрегация финансовых данных**
   - объединение дублей по `id`;
   - суммирование частей оплат и дополнительных работ;
   - обработка специальных значений `в ноль`, `стоп`, `end`.

4. **Формирование аналитической базы**
   - объединение таблиц по `id`;
   - использование поля `AM` из `prolongations.csv` как приоритетного менеджера;
   - исключение проектов, которые не должны участвовать в пролонгациях по правилам ТЗ.

5. **Расчёт коэффициентов пролонгации**
   - определение пролонгации в первый месяц после завершения проекта;
   - определение пролонгации во второй месяц только для проектов, не пролонгированных в первый;
   - расчёт коэффициентов по менеджерам и по отделу.

6. **Подготовка итогового отчёта**
   - формирование помесячных таблиц;
   - формирование годовых итогов;
   - экспорт результатов в Excel.
